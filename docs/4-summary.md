# 4.2 프로젝트 회고

## 잘 된 점

- 회원가입과 로그인부터 여행 일정 및 세부 일정 CRUD까지 주요 사용자 흐름을 하나의 웹 서비스로 연결하였다.
- 비밀번호 암호화, 세션 인증과 작성자·참여자 권한 검사를 적용하여 사용자 정보와 일정 변경 기능을 보호하였다.
- Kakao Map API와 Kakao Mobility API를 활용하여 장소 검색, 위치 마커, 이동 경로와 예상 이동시간을 제공하였다.
- 공유 링크, 참여자 등록과 `viewer`·`editor` 권한 관리를 구현하여 여러 사용자가 여행 일정을 함께 관리할 수 있도록 하였다.

---

# 4.2 프로젝트 회고

## 아쉬운 점 / 개선 사항

- 현재 공유 링크를 통해 일정에 참여할 수 있지만, 참여자가 스스로 해당 일정에서 탈퇴하는 기능은 구현되어 있지 않다. 초기 설계에서 참여 이후의 사용자 흐름을 충분히 고려하지 못해 참여 기능만 구현하고 탈퇴 기능을 놓친 점이 아쉽다.
- 향후 여행 상세 화면에 일정 탈퇴 기능을 추가하고, 탈퇴 시 `trip_members`에서 해당 참여 정보를 삭제하도록 개선할 필요가 있다.
- 향후 장소 순서 추천이나 날짜별 동선 최적화 기능을 추가하면 여행 계획의 완성도를 높일 수 있을 것 같다.
- 일정 공유와 권한 관리는 구현되었지만, 변경 내역을 확인할 수 있는 기능은 부족하다. 여러 사용자가 함께 수정할 때 누가 어떤 내용을 변경했는지 파악하기 어려운 점이 아쉽다. 향후 일정 수정 기록을 추가하면 공동 편집 과정의 신뢰성을 높일 수 있다.
