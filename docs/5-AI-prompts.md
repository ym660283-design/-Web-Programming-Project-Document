## 4. F-01 회원가입 화면 구현

날짜 : 2026년 06월 04일

모드 : Agent

지시사항 :

- GitHub Issue `[F-01] 회원가입 화면 및 기능 구현`과 프로젝트 문서를 확인하고, 프론트엔드 담당 범위인 회원가입 JSP 화면과 브라우저 입력 검증을 구현해줘.
- 회원가입 화면에는 아이디, 이름, 이메일, 비밀번호, 비밀번호 확인 입력 필드를 구성하고 기존 공통 헤더, 푸터, TripLog 디자인을 적용해줘.
- 폼은 백엔드 회원가입 Servlet과 연결할 수 있도록 `POST /register`로 제출하고, DB 설계와 동일한 필드명을 사용해줘.
- Bootstrap 유효성 검사 스타일과 별도 JavaScript 파일을 사용하여 필수 입력, 아이디 형식, 이메일 형식, 비밀번호 길이, 비밀번호 일치 여부를 안내해줘.

## 5. 여행 일정 목록 및 생성·수정 화면 구현

날짜 : 2026년 06월 06일

모드 : Agent

지시사항 :

- 예시 일정은 목적지, 기간, 제목, 설명, 상세보기, 수정 버튼을 포함하는 독립적인 카드 형태로 구성해줘.
- 여행 일정 생성과 수정은 별도의 JSP를 만들지 않고 기존 설계 구조에 맞게 공용 `views/trip/form.jsp`에서 처리해줘.
- 입력 폼에는 제목, 목적지, 시작일, 종료일, 설명 필드를 구성하고, 입력 요청이 `/trips` Servlet으로 전달되도록 연결해줘.
- `TripServlet`에서 목록, 생성, 수정, 상세보기 요청을 구분하여 각 JSP로 전달하고, DB 저장 기능 없이 화면 요청 전달까지만 구현해줘.
- 상세보기 화면에는 여행 제목, 목적지, 기간, 설명과 세부 일정 미등록 상태를 카드 형태로 표시하고 수정 화면으로 이동하는 버튼을 배치해줘.
